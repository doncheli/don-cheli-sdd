# /doncheli-debate

## Process
1. Accept decision, design, or proposal as input
2. @doncheli-debate — Instantiate adversarial roles:
   - CPO: business value, user impact, time-to-market
   - Architect: technical soundness, scalability, maintainability
   - QA Lead: testability, edge cases, failure modes
   - Security: threat surface, OWASP exposure, data handling
3. Round 1 — Each role presents position independently
4. Round 2 — Cross-examination: each role challenges the others
5. Round 3 — Synthesis: identify points of consensus and unresolved conflicts
6. Wait for user to resolve open conflicts or provide additional context
7. Output final recommendation with dissenting notes and risk register
