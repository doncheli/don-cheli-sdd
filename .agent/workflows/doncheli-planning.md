# /doncheli-planning

## Process
1. Accept backlog items, open RFCs, and team capacity as input
2. @doncheli-planning — Phase 1: RFC Review
   - Summarize open RFCs and their current status
   - Flag RFCs ready for decision vs. still in discussion
   - Present blocking dependencies between RFCs
3. Phase 2: WSJF Prioritization
   - Score each backlog item: Cost of Delay / Job Duration
   - Cost of Delay = User-Business Value + Time Criticality + Risk Reduction
   - Sort by WSJF score descending
4. Wait for user to adjust scores or override priorities
5. Phase 3: Capacity Matching
   - Map prioritized items to available squad members
   - Flag over-allocation and propose re-balancing
   - Identify items that need splitting to fit within sprint capacity
6. Phase 4: Commitment
   - Produce sprint commitment list with owner, story points, and definition of done
   - Generate dependency graph for parallel work streams
7. Output weekly planning document: committed items, parking lot, and risk register
