EL PROBLEMA

El "Vibe Coding" esta roto

Cuando usas IA para programar sin estructura, ocurren tres cosas terribles:

🧠

Amnesia entre sesiones

La IA olvida las decisiones de arquitectura que tomaron ayer. Cada sesion nueva es empezar de cero. Tu contexto se pierde como lagrimas en la lluvia.

👻

Stubs silenciosos

La IA dice "ya implemente X" pero el codigo tiene // TODO: implementar. Sin verificacion, estos fantasmas pasan desapercibidos hasta produccion.

🎲

Zero verificacion

No hay evidencia de que nada funcione. No hay tests, no hay specs, no hay quality gates. Solo esperanza y vibes.

✨ Don Cheli resuelve esto

Con un pipeline estructurado de 6 fases, quality gates automaticos, y persistencia entre sesiones. Tu IA pasa de improvisar a hacer ingenieria real.

COMO FUNCIONA

6 fases. De la idea al codigo verificado.

Don Cheli guia a tu IA paso a paso, como un director tecnico senior.

💡 Idea → 📄 Spec → 🔍 Clarificar → 🛠 Plan → ⚡ Codigo → ✓ Revisado
1

Especificar

/dc:especificar

Describes tu idea en lenguaje natural y Don Cheli la convierte en una especificacion Gherkin precisa con escenarios de prueba, prioridades y un esquema de datos provisional.

2

Clarificar

/dc:clarificar

Un QA virtual revisa tu especificacion buscando ambiguedades y contradicciones. Todo lo que no esta claro se resuelve aqui, no en produccion.

3

Planificar

/dc:planificar-tecnico

Se genera un blueprint tecnico con la arquitectura, los contratos de API y el esquema de datos final. Como un plano de construccion antes de poner ladrillos.

4

Desglosar

/dc:desglosar

El plan se divide en tareas concretas con un orden de ejecucion claro. Cada tarea sabe exactamente que archivos toca y que tests necesita.

5

Implementar

/dc:implementar

La IA escribe el codigo siguiendo TDD estricto: primero el test (que falla), luego el codigo (que lo hace pasar), despues mejora. Nada pasa sin pruebas.

6

Revisar

/dc:revisar

Peer review automatico en 7 dimensiones: funcionalidad, tests, rendimiento, arquitectura, seguridad, mantenibilidad y documentacion.

NO NEGOCIABLE

Las 3 Leyes de Hierro

Estos principios jamas se rompen. Son la base de todo lo que Don Cheli hace.

LEY I

⚡ TDD Obligatorio

Todo codigo de produccion nace de un test. Sin excepciones. Primero escribes la prueba, luego el codigo.

RED → GREEN → REFACTOR
LEY II

🔍 Causa Raiz Primero

Cuando algo falla, no se parchea. Se investiga la causa real siguiendo un protocolo estricto de 5 pasos.

Reproducir → Aislar → Corregir
LEY III

✔ Evidencia, No Promesas

Nada se da por hecho sin pruebas. "Creo que funciona" no existe. Solo valen capturas, logs y tests verdes.

✘ "Creo que funciona" ✔ Tests pasan (evidencia)

RAZONAMIENTO

15 modelos mentales integrados

No solo codigo. Don Cheli te ayuda a pensar mejor con modelos mentales de clase mundial.

🧰

Primeros Principios

Descomponer hasta las verdades fundamentales y reconstruir desde cero.

❓

5 Porques

Causa raiz: pregunta "por que?" hasta llegar al origen real del problema.

📈

Pareto 80/20

Enfocate en el 20% que produce el 80% de los resultados.

🔄

Inversion

En vez de como lograr X, pregunta: como GARANTIZO el fracaso?

🎲

Pre-Mortem

Imagina que ya fallo. Analiza por que. Previene en vez de lamentar.

💡

Segundo Orden

Consecuencias de las consecuencias. Piensa 2-3 pasos mas alla.

⚖

Costo de Oportunidad

Que sacrificas al elegir una opcion? Toda decision tiene un costo oculto.

🗺

Mapa vs Territorio

Distingue entre tu modelo mental y la realidad. El mapa NO es el territorio.

🎯

Minimizar Arrepentimiento

Framework de Jeff Bezos: que decision minimiza el arrepentimiento a largo plazo?

💰

Reversibilidad

Calibra el compromiso: las decisiones reversibles se toman rapido, las irreversibles con cuidado.

🎞

Circulo de Competencia

Conoce tus limites. Distingue lo que dominas de lo que solo conoces superficialmente.

🎲

Probabilistico

Razona en probabilidades, no en certezas binarias. Asigna % a los resultados.

🤖

RLM: Verificacion

Sub-LLMs frescos como jurado independiente para verificar conclusiones.

🔗

RLM: Cadena

Context Folding: subagentes frescos por paso para evitar degradacion del contexto.

🔨

RLM: Descomposicion

Divide y conquista con subagentes paralelos para problemas complejos.

INSTALACION

En 3 pasos. 2 minutos.

Gratis, open source, sin dependencias pesadas. Solo necesitas Git y un agente de IA.

1

Clona el repo

Descarga Don Cheli desde GitHub

2

Ejecuta el instalador

El instalador interactivo te guia paso a paso

3

Empieza a usar

Escribe /dc:comenzar en tu agente de IA

# 1. Clonar git clone https://github.com/doncheli/don-cheli-sdd.git   # 2. Instalar (interactivo: elige idioma, herramienta y perfil) cd don-cheli-sdd && bash scripts/instalar.sh   # 3. En tu proyecto, abre tu agente IA y escribe: /dc:iniciar

O instalacion remota en una sola linea:
