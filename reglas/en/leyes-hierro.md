# Iron Laws

## Non-Negotiable. Always apply.

### 1. TDD Law
All production code REQUIRES tests.
- `RED` → Write a failing test
- `GREEN` → Minimum code to pass
- `REFACTOR` → Improve without changing behavior

### 2. Debugging Law
Root cause first, then the fix.
1. REPRODUCE → ISOLATE → UNDERSTAND → FIX → VERIFY

### 3. Verification Law
Evidence before assertions.
- ✅ "Tests pass (evidence attached)"
- ❌ "I think it works"
